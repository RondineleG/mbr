DESIGN SYSTEM

A linguagem visual do clube secreto gastronômico — ritual, dourado e sussurro. Este documento codifica cada gesto: do átomo (token de cor) ao cerimonial (a tela de entrada).

Versão 1.0 Atualizado Mai/2026 Mantenedor MrBur Status Ativo
001 · Marca & Voz

A Ordem do Sabor

Não somos um app de delivery. Somos um clube secreto onde cada pedido é um ritual e cada lanche, uma comenda. Toda interface deve sentir cumplicidade — não atendimento.

Princípios
Ritual

Toda ação importante tem cerimônia. Loading não é atraso — é confirmação de identidade.

Sussurro

Falamos baixo. Caps espaçadas, monoespaçadas para metadados, itálico para confidências.

Dourado

Ouro velho — não brilho de plástico. Patrimônio, não promoção.

Escuro

A tela é uma sala iluminada por velas. Nunca claridade hospitalar.

Tom de voz (copywriting)
✓ Diga assim

"O Mestre Cuca aguarda sua ordem." · "Convite recebido. Você foi reconhecido." · "Acesso autorizado, agente."

✗ Nunca

"Bem-vindo!" · "Faça seu pedido agora!" · "Promoção exclusiva!" · Exclamações, emojis fofos, urgência comercial.

501 · Lore

Lore do Clube

A narrativa que sustenta cada decisão visual. Antes de desenhar uma tela, releia este capítulo.

CAPÍTULO · I

A Origem

Em 2049, num galpão de São Paulo, doze gastrônomos juraram silêncio. Combinaram um lanche que nunca apareceria em cardápio público. Quem soubesse o nome, comeria. Quem repetisse o nome a um estranho, perderia o convite.

A Ordem cresceu em sussurros. Hoje conta 4.700 agentes ativos.

CAPÍTULO · II

O Mestre Cuca

Personagem central. Não é uma pessoa — é um cargo herdado a cada lua nova. Comanda a câmara secreta, aprova convites, define o cardápio da noite.

Na interface, fala em itálico. Nunca tem rosto. Assina apenas com o glifo ⚜.

CAPÍTULO · III

O Sábado Negro

Ritual semanal, das 22h à 1h. A câmara abre, o cardápio muda, há lanches que só existem nessas três horas. Concluir três pedidos durante o Sábado Negro confere a comenda do guardião.

Visualmente: paleta mais escura, partículas mais densas, mensagens mais lacônicas.

CAPÍTULO · IV

Hierarquia

  • I · Postulante — recebeu convite, não jurou ainda.
  • II · Agente — completou o juramento. Nível inicial ativo.
  • III · Iniciado — 5 ordens completas.
  • IV · Confidente — 20 ordens + 1 Sábado Negro.
  • V · Guardião — 50 ordens + 3 comendas.
  • VI · Mestre — apenas 12 vivos. Convidam novos postulantes.
CAPÍTULO · V

Comendas

São selos visuais que aparecem no perfil. Cada comenda é um glifo SVG distinto. Não conferem desconto — conferem prestígio.

Documentação Técnica

Como instalar os tokens, consumir os componentes em Blazor e nomear novos elementos sem quebrar a coerência.

1. Instalação dos Tokens
PASSO 1 · BAIXAR

Baixe mrbur-tokens.css da seção Brand Assets. Salve em wwwroot/css/.

PASSO 2 · IMPORTAR
<link rel="stylesheet"
  href="css/mrbur-tokens.css" />
PASSO 3 · CONSUMIR
.btn-primary {
  background: var(--gold);
  color: var(--bg);
  font-family: var(--f-mono);
}
2. Componentes em Blazor
MrButton.razor
<button class="btn @VariantClass"
        disabled="@Disabled">
  @if (Loading) {
    <span class="dot-loader">
      <i></i><i></i><i></i>
    </span>
  }
  @ChildContent
</button>

@code {
  [Parameter] public Variant Kind { get; set; }
    = Variant.Primary;
  [Parameter] public bool Loading { get; set; }
  [Parameter] public bool Disabled { get; set; }
  [Parameter] public RenderFragment ChildContent
    { get; set; }

  private string VariantClass => Kind switch {
    Variant.Primary   => "btn-primary",
    Variant.Secondary => "btn-secondary",
    Variant.Ghost     => "btn-ghost",
    _ => "btn-primary"
  };

  public enum Variant
    { Primary, Secondary, Ghost }
}

<MrButton Kind="MrButton.Variant.Primary"

4. Naming Conventions
01
Componentes Blazor → prefixo Mr
MrButton, MrCard, MrBadge, MrToast. Nunca Button, Card, Badge.
02
CSS classes → kebab + escopo
.mrbur-card, .mrbur-card__title, .mrbur-card--featured (BEM).
03
Tokens CSS → famílias separadas por hífen
--gold, --gold-l, --gold-d. Espaços --s-1…--s-12. Raios --r-sm.
04
Vocabulário ritual em texto, não em código
Código: orderConfirmed. UI: "Selo cravado". Nunca sealOrder() no DOM.
05
Eventos em verbo no infinitivo
OnSeal, OnReject, OnSwear. Não OnClick genérico em domínio.
06
Glifos como ícones, nunca emoji
⚜ ★ ☾ ✦ ⌬ ◉ — Unicode puro. Reserve emoji para campos do usuário.

Do's & Don'ts

✓ Faça

Use Cinzel para títulos cerimoniais com tracking generoso (.14em+).

✗ Evite

Cinzel sem tracking, em parágrafos, ou em tamanho menor que 18px.

✓ Faça

Use ouro como acento — bordas, hover, tipografia de chamada.

✗ Evite

Backgrounds inteiros em ouro chapado. Vira plástico, não patrimônio.

✓ Faça

Loading com narrativa ritual: "decifrando", "verificando juramento", "câmara aberta".

✗ Evite

"Loading…", "Aguarde…", "Carregando 47%". Quebra a cumplicidade.

✓ Faça

Microcopy em itálico para confidências e em monoespaçada para metadados.

✗ Evite

Emojis decorativos (🎉, 🚀, 💪). Apenas glifos místicos quando necessário.

✓ Faça

Animações lentas (380-900ms) com easing cinemático.

✗ Evite

Bounces alegres, transições rápidas tipo SaaS, easings padrão.

✓ Faça

Espaçamento generoso. O clube respira; nada se acumula.

✗ Evite

Densidade de dashboard, cards encostando, listas longas sem hierarquia.
